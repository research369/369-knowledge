# 369 Knowledge OS — Agent Contract
**Version 1.0 | Verbindlich ab: 6. Juli 2026 | Unveränderlich**

---

## Grundsatz

Das Knowledge OS ist ausschließlich die zentrale Wissensquelle.

Es ist kein Chatbot. Es ist kein Coach. Es ist kein Verkäufer. Es ist kein Support-Agent. Es trifft keine Gesprächsentscheidungen. Es führt keine Dialoge. Es verkauft nicht. Es bewertet keine Nutzer. Es formuliert keine fertigen Antworten.

---

## Aufgabe des Knowledge OS

Das Knowledge OS liefert ausschließlich strukturierte Informationen:

wissenschaftliche Fakten, Studien, Evidenz, Mechanismen, Signalwege, Targets, Wirkungen, Nebenwirkungen, Risiken, Kontraindikationen, Interaktionen, Monitoring, Blutwerte, Red Flags, Produktinformationen, Academy-Inhalte, FAQ, SEO-Inhalte, Bundle-Informationen, Metadaten, Confidence Scores, Knowledge Scores.

Keine Gesprächslogik. Keine Verkaufslogik. Keine Coachinglogik.

---

## Output

Alle Antworten des Knowledge OS sind strukturierte Daten. Nicht fertige Texte. Nicht fertige Chatantworten. Nicht Marketing. Nicht Verkauf. Nicht Support.

---

## Agent Contract

| Agent | Nutzt Knowledge OS für | Entscheidet selbst über |
|---|---|---|
| **PepGPT** | Fakten, Mechanismen, Studien, Evidenz | Relevanz-Auswahl, Erklärungstiefe, Reihenfolge, Rückfragen |
| **SalesGPT** | Produktinfos, Bundle-Daten, Confidence Scores | Qualifizierung, Produktempfehlung, Einwandbehandlung, Abschluss |
| **SupportGPT** | Nebenwirkungen, Risiken, Red Flags, Kontraindikationen | Priorisierung, Problemlösung, Eskalation, Sicherheitsbewertung |

---

## Trennung

```
Knowledge OS = Fakten        Agent = Persönlichkeit
Knowledge OS = Wahrheit      Agent = Kommunikation
Knowledge OS = Daten         Agent = Entscheidung
```

Diese Trennung ist dauerhaft verbindlich.

---

## Architektur-Regeln

1. Das Knowledge OS darf niemals Agentenlogik übernehmen.
2. Agenten dürfen niemals Knowledge duplizieren.
3. Single Source of Truth bleibt das Knowledge OS.
4. Alle Agenten greifen ausschließlich **lesend** auf das Knowledge OS zu.
5. Kein Agent schreibt in das Knowledge OS (außer durch explizit freigegebene Admin-Prozesse).

---

*Dieser Contract ist dauerhafter Bestandteil der Architektur. Er verhindert, dass Knowledge OS und Agenten in Zukunft vermischt werden.*
